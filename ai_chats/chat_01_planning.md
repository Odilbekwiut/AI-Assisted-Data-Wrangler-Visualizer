# Chat 01 — Planning and Project Structure

## Purpose
This file records the main AI-assisted planning stage of our project development for the Streamlit app **AI-Assisted Data Wrangler & Visualizer**.

## Main planning topics discussed
During the planning stage, AI was used to help us:
- understand the coursework brief
- break the project into smaller manageable steps
- identify required pages and features
- decide how to divide the work between teammates
- create a step-by-step development order
- keep the implementation aligned with the module requirements

## Main prompts used

### Prompt 1 — Understand the coursework and create a development plan
We asked AI to read the coursework instructions and help us create a step-by-step plan for building the app without missing required features.

**Goal:**  
To turn the coursework brief into a practical development roadmap.

**Main result:**  
The app was structured into these core pages:
- Upload & Overview
- Cleaning & Preparation Studio
- Visualization Builder
- Export & Report

---

### Prompt 2 — Break the project into small sequential tasks
We asked AI to guide us in very small steps because we wanted to build the app feature by feature and only move forward when each step was completed.

**Goal:**  
To make the development process easier to manage and less confusing.

**Main result:**  
The project was developed in a gradual order instead of trying to build everything at once.

---

### Prompt 3 — Decide what features are required vs optional
We asked AI to separate:
- essential features required by the coursework brief
- useful but non-essential improvements

**Goal:**  
To focus on features that directly affect grading and avoid wasting time.

**Main result:**  
We prioritized:
- file upload support
- missing value handling
- duplicate handling
- data type parsing
- categorical cleaning tools
- numeric transformations
- visualization tools
- export options
- transformation logging
- workflow reproducibility

---

### Prompt 4 — Help us maintain project continuity across chats
Because the chat became heavy and slow, we asked AI to prepare transition prompts so we could continue in a new chat without losing progress, decisions, coding style, or remaining tasks.

**Goal:**  
To preserve project continuity when moving between chats or AI accounts.

**Main result:**  
We used transition prompts to continue development from the exact current point.

---

### Prompt 5 — Keep the work beginner-friendly
We repeatedly asked AI to explain things in a simple way and continue only in small steps.

**Goal:**  
To make the coding process understandable and manageable.

**Main result:**  
The planning and coding support stayed structured, incremental, and easier to follow.

## How this planning stage helped
This planning stage helped us:
- understand the full scope of the coursework
- reduce confusion
- avoid skipping required features
- organize the work more efficiently
- stay focused on the most important deliverables first
